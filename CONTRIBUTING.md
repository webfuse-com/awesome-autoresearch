# Contributing

Thanks for helping improve this list.

## What to add

Open a PR if a repo:

- is directly inspired by `karpathy/autoresearch`,
- explicitly cites autoresearch as a foundation,
- or clearly uses the same autonomous improvement loop in a meaningful adjacent domain.

## Preferred entry format

Use this format:

```md
- [owner/repo](https://github.com/owner/repo) - Short, neutral description.
```

Keep descriptions:

- concise,
- factual,
- non-promotional,
- consistent with the rest of the list.

## Good PRs

- add strong missing repos,
- improve category placement,
- tighten wording,
- remove weak fits or duplicates,
- add small metadata improvements without clutter.

## Please avoid

- marketing copy,
- keyword stuffing,
- adding loosely related “AI research” repos with no autoresearch connection,
- very long descriptions,
- broad unrelated sections.

## Curation standard

This is a curated list, not an exhaustive search dump.

If a project is adjacent but not clearly autoresearch-related, it may still fit under an “Adjacent systems” section, but the connection should be obvious.
